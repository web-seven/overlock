A registry-focused release: the local registry now speaks HTTPS, gains a `load-image` subcommand, and several rough edges around environment and registry creation are smoother.

<!-- truncate -->

## What's new

- **HTTPS for the local registry** ([#346](https://github.com/web-seven/overlock/pull/346)) — local registries now serve HTTPS in addition to HTTP, fronted by an nginx reverse proxy ([#347](https://github.com/web-seven/overlock/pull/347)).
- **`registry load-image` subcommand** ([#342](https://github.com/web-seven/overlock/pull/342)) — push pre-built OCI images straight into the local registry without going through `docker push`.
  - `--helm` flag to load Helm charts ([#345](https://github.com/web-seven/overlock/pull/345)).
  - Always uses the OCI empty base layer ([#349](https://github.com/web-seven/overlock/pull/349)).
- **`--container-path` on `environment create`** ([#354](https://github.com/web-seven/overlock/pull/354)) — pin a custom container path for environment data.
- **`--version` on the install script** ([#356](https://github.com/web-seven/overlock/pull/356)) — install a specific Overlock version.
- **Optional admin service account creation** ([#338](https://github.com/web-seven/overlock/pull/338)).
- **README refresh** with badges and clearer navigation ([#367](https://github.com/web-seven/overlock/pull/367)).

## Behaviour changes

- **`registry create` warns instead of erroring on duplicates** ([#352](https://github.com/web-seven/overlock/pull/352)).
- **`environment create` warns instead of erroring on duplicate names** ([#336](https://github.com/web-seven/overlock/pull/336)).

## Bug fixes

- **Kyverno policy CRD conflicts** resolved ([#340](https://github.com/web-seven/overlock/pull/340)).
